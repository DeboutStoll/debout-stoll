§1 · Vue d'ensemble & prérequis

Le site est un site statique : des fichiers HTML/CSS/JS publiés sur le réseau mondial de GitHub Pages. Il n'y a pas de serveur applicatif — les parties « dynamiques » sont déléguées à des services externes.

ÉlémentFonctionnement en production
Le site (pages, FR/EN, PWA, animations)HTML statique servi par GitHub Pages (CDN mondial, HTTPS automatique)
Formulaire d'adhésionZoho Form intégré → notification à contact@debout-stoll.com
Formulaire de contribution (+ photo)Zoho Form intégré (upload de fichier) → modération dans Zoho
Compteur d'adhésionsSupabase lu côté navigateur via la clé publique anon

Comptes à préparer

  • GitHub — accès au compte DeboutStoll (dépôt DeboutStoll/debout-stoll).
  • Namecheap — accès au DNS de debout-stoll.com (onglet Advanced DNS).
  • Zoho — un compte Zoho Forms (le même que Zoho Mail de contact@debout-stoll.com).
  • Supabase — un projet (facultatif, uniquement pour le compteur en direct).
Pourquoi c'est accessible partout dans le monde : GitHub Pages sert le site depuis un réseau de diffusion (CDN) réparti sur toute la planète. Un visiteur à Douala, Paris ou Montréal reçoit les pages en quelques millisecondes, sans calcul serveur. Le site est aussi installable comme application (PWA) et dispose d'un mode « data-light » pour les connexions lentes.
État actuel : tout le code est prêt sur la branche github-pages du dépôt. Ce guide vous mène de cette branche jusqu'au site en ligne. La branche main (ancienne version) reste intacte tant que vous ne fusionnez pas.

Sécurité — à faire en premier

Deux réflexes indispensables avant de commencer.

1

Renouveler le token GitHub

Un token d'accès personnel (PAT) a été partagé en clair : il doit être considéré comme compromis.

  • Ouvrez https://github.com/settings/tokens → supprimez l'ancien token.
  • Créez-en un nouveau (Fine-grained token), limité au dépôt debout-stoll, permission Contents : Read and write, expiration courte.
  • Ne le collez jamais dans un chat, un fichier versionné ou une capture d'écran.
2

Protéger les secrets

Ne partagez jamais :

  • votre mot de passe de compte (GitHub, Supabase, Zoho) — aucun service n'en a besoin dans le site ;
  • la clé service_role de Supabase (secrète). Le site n'utilise que la clé anon, qui est publique par nature.

Les clés se rangent uniquement dans GitHub → Settings → Secrets and variables → Actions (voir Phase 4).

Compte GitHub : utilisez toujours DeboutStoll (email contact@debout-stoll.com), jamais infos-stratedged. La configuration git de la machine a déjà été corrigée dans ce sens.

1

Créer les formulaires

Sur https://www.zoho.com/forms, créez les deux formulaires ci-dessus.

2

Configurer les notifications

Pour chaque formulaire : Settings → Email Notifications → destinataire contact@debout-stoll.com. Activez au besoin un email d'accusé de réception vers l'adresse du visiteur.

3

Récupérer les liens d'intégration

Pour chaque formulaire : Share → Embed → copiez l'URL Permalink (https://forms.zohopublic.com/…/formperma/…).

4

Coller les liens dans le site

Éditez le fichier content/forms.ts :

export const zohoForms = {
  join:       'https://forms.zohopublic.com/…/formperma/…',
  contribute: 'https://forms.zohopublic.com/…/formperma/…',
};

Puis git add content/forms.ts && git commit -m "Zoho form links" && git push. Les formulaires apparaissent sur le site après le redéploiement (~2 min).

Supabase — le compteur d'adhésions

Facultatif mais demandé. Le navigateur lit directement un simple nombre depuis Supabase — jamais les données des membres.

1

Créer le projet & les tables

Sur https://supabase.com : New project (région Europe conseillée). Puis SQL Editor → exécutez le fichier supabase/schema.sql, puis supabase/member-count.sql.

2

Récupérer les identifiants publics

Settings → API → copiez :

  • le Project URL ;
  • la clé anon public (celle-ci est sûre à exposer — pas la clé service_role).
3

Renseigner les secrets GitHub

Dépôt GitHub → Settings → Secrets and variables → Actions :

NomTypeValeur
NEXT_PUBLIC_SUPABASE_URLSecretle Project URL
NEXT_PUBLIC_SUPABASE_ANON_KEYSecretla clé anon public
NEXT_PUBLIC_MEMBER_BASE_COUNTVariablepoint de départ (ex. 250 ou 0)
4

Alimenter le compteur (choisir une option)

  • Webhook Zoho → Supabase (recommandé) : dans Zoho Forms, Integrations → Webhook insère une ligne dans members à chaque adhésion. Le compteur reflète alors les vrais membres.
  • Manuel : mettez à jour NEXT_PUBLIC_MEMBER_BASE_COUNT de temps en temps d'après le total Zoho.

Si vous n'utilisez pas Supabase, le compteur affiche simplement le nombre de base.

GitHub Pages — publier le site

1

Activer Pages

Dépôt → Settings → Pages → Build and deployment → Source = « GitHub Actions ».

2

Déclencher le premier déploiement

Le workflow .github/workflows/deploy.yml publie à chaque push sur main. Fusionnez la branche prête :

git checkout main
git merge github-pages
git push
3

Suivre la publication

Onglet Actions → le job « Deploy to GitHub Pages » doit passer au vert (~2 min). Le site est alors visible sur https://deboutstoll.github.io en attendant le domaine.

Astuce : attendez d'avoir collé les liens Zoho (Phase 2) et renseigné les secrets Supabase (Phase 3) avant de fusionner, pour que la première mise en ligne soit déjà complète.

Namecheap DNS — brancher le domaine

1

Déclarer le domaine côté GitHub

Dépôt → Settings → Pages → Custom domain → saisissez debout-stoll.com → Save. (Le fichier public/CNAME le fixe déjà aussi.)

2

Configurer le DNS chez Namecheap

Domain List → Manage → Advanced DNS. Supprimez le CNAME www → parkingpage et l'URL Redirect @ par défaut, puis ajoutez :

TypeHostValueTTL
A Record@185.199.108.153Automatic
A Record@185.199.109.153Automatic
A Record@185.199.110.153Automatic
A Record@185.199.111.153Automatic
CNAME Recordwwwdeboutstoll.github.io.Automatic
Ne touchez pas aux enregistrements Zoho Mail (MX / SPF / DKIM) de contact@debout-stoll.com : ils sont indépendants de GitHub Pages et doivent rester en place.
3

Activer HTTPS

Après propagation (10–60 min), GitHub Settings → Pages affiche le domaine vérifié → cochez Enforce HTTPS. Le certificat est délivré automatiquement.

Vérification — liste de contrôle

Après propagation DNS, contrôlez chaque point.

  • https://debout-stoll.com se charge en HTTPS et redirige vers /fr.
  • www.debout-stoll.com fonctionne ; /en fonctionne ; le sélecteur FR|EN conserve la page.
  • Test sur téléphone réel (portrait/paysage), tablette et ordinateur — aucun débordement, boutons cliquables.
  • Le formulaire d'adhésion s'affiche, s'envoie, et un email arrive à contact@debout-stoll.com.
  • Le formulaire de contribution accepte un fichier ; l'envoi est reçu.
  • Le compteur affiche un nombre (de base, ou en direct si Supabase est alimenté).
  • L'application est installable (PWA) ; le mode hors-ligne fonctionne sur les pages visitées.
  • sitemap.xml et robots.txt sont accessibles et en .com.
  • Aperçu correct sur WhatsApp / Facebook / LinkedIn (Open Graph).
dig +short debout-stoll.com A      # → les quatre 185.199.108–111.153
curl -sI https://debout-stoll.com/  # → 200/redirection, HTTPS

Annexe A — Comptes GitHub & JetBrains

La machine authentifie désormais git en tant que DeboutStoll (contact@debout-stoll.com), sans invite de connexion. Pour que l'IDE JetBrains cesse d'ouvrir la page d'autorisation GitHub :

  • Settings → Version Control → GitHub → retirer le compte infos-stratedged → Add account → Log In with Token → coller le token DeboutStoll.
  • Settings → Version Control → Git → activer « Use credential helper ».
